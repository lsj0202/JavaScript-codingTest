## 자료구조?

자료(data)를 담기위한 구조
데이터의 수가 많아질 수록 더 효율적인 자료구조가 필요하다.

자료구조를 제대로 활용하지 못하면 불필요한 데이터를 낭비할 수 있다.

### 선형 구조

하나의 데이터 뒤에 다른 데이터가 연결 되어있다. (일렬)

- 배열
- 연결 리스트
- 스택
- 큐

### 비선형 구조

하나의 데이터 뒤에 여러개의 데이터가 올 수 있다.

- 트리
- 그래프

## 프로그램 성능 측정 방법

- 시간 복잡도: 연산 횟수
- 공간 복잡도: 메모리의 양

#### Big - O

1. 특정 알고리즘이 얼마나 효율적인지 수치적으로 표현
2. 가장 빠르게 증가하는 항만 고려
3. 가장 큰 항에 붙어있는 계수는 제거

- On, nlogn, On^2, On^3
